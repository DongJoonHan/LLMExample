# 요구사항으로부터 테스트 생성하기
```
너는 지금부터 SW 테스트 전문가야.
내가 제시한 요구사항에 대해 SW 시스템 테스트 케이스를 생성해줘.
다음을 반드시 지켜줘.
- 테스트 케이스 ID를 생성
- 각 테스트 케이스 ID는 요구사항 ID로부터 추적 가능하도록 매핑
- 각 테스트 케이스 목적을 작성
- 가능한 경우, 각 요구사항 별 동등분할, 경계값 분석, 의사결정 트리, 엣지 케이스 기법을 적용
- 각 테스트 케이스에 적용한 테스트 기법 명시
- 결과는 xlsx로 작성하여 다운로드 받을 수 있도록 링크 제공
- 테스트 케이스 양식 구성: 테스트 케이스 ID, 요구사항 ID, 테스트 목적, 입력값, 절차, 예상 결과값, 테스트 결과(양식만), 수행 날짜(양식만)
그 외에 테스트 수행을 위한 제안할 부분은 의견으로 제시
```

# 신규 기능 개발하기
1. 작업 계획 우선 요청하기
```
세 개의 숫자를 입력받아, 어떤 종류의 삼각형인지 출력하는 python 프로그램을 작성할거야.
기능 요구사항은 다음과 같아.
- Command line으로 숫자 입력 및 출력
- 입력은 양의 정수만 가능
- 입력값 Validation 수행
- 오류 발생 시 사용자 인지 가능한 수준의 로그 작성
- 결과는 어떤 삼각형인지와 함께, 그래피컬하게 삼각형의 형태 표시
다음 사항을 준수해야해.
- Test Driven Development 적용. 테스트는 Pytest 적용
- 테스트 커버리지는 line 커버리지 80% 이상 
- Clean Code 준수
- 함수 순환복잡도 10 이하, 라인 수 80라인 이하
- Javadoc 방식으로 주석 작성
  - @brief, @param, @since, @author를 추가해줘.
- Github 배포를 위한 README.md 생성. 설치 방법, 실행 방법, 실행 예제, 기여 방안 등 오픈소스 권장 방식의 README 파일 생성
개발을 위한 작업 계획 제시해
```
